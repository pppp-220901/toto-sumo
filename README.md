# トト相撲 ONLINE

大相撲の取組を予想して「持ち米」を賭ける、ブラウザだけで遊べる無料の相撲予想ゲームです。サーバー不要・単一HTMLファイルで動作し、[GitHub Pages](https://pages.github.com/) にそのまま公開できます。

## 特徴

### 本場所モード（初日〜千秋楽）

- **実データ運用（手動登録）**: 実際の大相撲の取組・勝敗・決まり手・優勝力士・三賞で遊べます。各サイトの利用規約に配慮して自動スクレイピングは行わず、運営が「取組結果登録ツール」でABEMA TIMESの情報を手動登録します（データが無い場合はシミュレーションモードにフォールバック）
- **多彩な賭け方**: 取組ごとの勝敗予想を軸に、決まり手予想（同時予想で上乗せ配当）、東西対抗戦（東西の白星の差が、そのまま配当倍率に）、優勝予想（当たれば千秋楽の持ち米が倍付け・早く張るほど高倍率）、三賞予想（殊勲・敢闘・技能）に対応
- **オッズの駆け引き**: オッズは番付と調子で毎日変動。倍付けボーナスはすべて「文脈に紐づいた」設計で、所属一門×1.5・推し力士×1.5（重複で×2.25）、🌅初白星（まだ0勝の力士）×2、☆勝ち越し王手×2、🩹怪我明け×2、🚩懸賞一番は誰でも×3。一門・推しは中日まで「変化」（変更）できます
- **みんなの賭け動向**: 場所予想タブに、参加者全員の賭け件数を合算した力士人気ランキングを表示。人気に乗るか、逆張りするか
- **ちゃんこ・相撲クイズ**: 1日1回のちゃんこと1日1問のクイズ（全45問）で、本場所中は毎日無料で持ち米を増やせます
- **不戦勝・不戦敗の自動返還**: 力士の休場による不戦は的中/不的中の対象外とし、賭け金を全額自動返還します

### オフシーズンモード（千秋楽後〜次の初日）

- **総合発表**: 千秋楽3日後の水曜10:00に「トト番付編成会議」が開かれ、来場所の番付が内定。番付に応じた編成会議ボーナス（トト綱+15,000〜関取+3,000合）が、来場所の初期持ち米（基本15,000合）に上乗せされます
- **番付予想モード**: 来場所の大相撲の新番付・上位16人（横綱〜前頭上位）をピタリ予想。三役の人数は1〜4人で調整でき、常に合計16人になるよう前頭側の範囲が自動で伸縮します。全問的中で+30,000合、1ミス+15,000合、以降1ミスごとに-1,000合。締切（番付発表の前日）までは確定後も何度でも修正可能
- **答え合わせ**: 番付発表（本場所2週間前の月曜）後に運営が新番付を登録すると、自動採点・ボーナス加算されます

### 番付・進行システム

- **トト番付**: トト綱・トー関・トト脇・ト結・関取・幕下・新弟子の7段階。千秋楽の持ち米で毎場所編成されます。1場所目は全員が新弟子、2場所目以降は最低でも幕下（新弟子には戻りません）
- **毎場所リスタート経済**: 持ち米は毎場所「基本15,000合＋編成会議ボーナス＋番付予想ボーナス」から再スタート。古参の逃げ切りを防ぎ、毎場所が真剣勝負になります
- **称号・実績**: 95種類の称号を収集。称号は四股名の前に表示され、番付表でタップすると「ひとこと」も見られます
- **過去のトト番付アーカイブ**: 終了した場所の番付・成績をいつでも振り返れます

### 使いやすさ・共有

- **チュートリアル**: 初めて本場所に入るとスワイプ式の遊び方ガイド（6スライド）を自動表示。「？」ボタンでいつでも読み返せます
- **見やすさ優先モード**: ヘッダーの「Aa」ボタンで、ゴシック体・大きめ文字・高コントラストの表示に切り替えられます
- **画像で共有**: 番付予想と1日のリザルト（収支・的中率・賭けの内訳）を番付表風のPNG画像として保存できます。iPhoneのSafariでは共有シートから「写真」に直接保存可能。#トト相撲
- **ログインID・パスワード認証**: 実在するメールアドレスは一切収集しません。パスワードはSupabase Auth側でハッシュ化され、運営者を含め誰も元のパスワードを見ることができない設計です（詳細は[ログイン認証仕様.md](./ログイン認証仕様.md)）
- **クロスデバイス同期**: ログインIDを設定すれば、持ち米・称号・推し力士・的中履歴・実績・未精算ベットまで、別の端末からも続きから遊べます
- **サーバー不要の共有ランキング**: 取組結果は日付から決まるシード乱数で全プレイヤー共通に生成されるため、賭けの精算だけをSupabaseで共有すれば、友達同士で同じ番付・同じランキングを争えます

## 遊び方

1. 入門の儀で四股名・所属一門・推し力士を決める（チュートリアルが案内します）
2. 「取組予想」タブで各取組に持ち米を賭ける（締切は取組ごとに個別）
3. 「場所予想」タブで優勝力士・三賞を予想。みんなの賭け動向もここでチェック
4. 結果は毎日自動精算。ログインすればまとめてポップアップで確認できます
5. 千秋楽の持ち米で来場所のトト番付が決まり、水曜10:00の総合発表で内定
6. オフシーズンは「番付予想」で来場所の新番付を予想して待つ

Supabaseプロジェクトの作成からGitHub Pagesでの公開、実データ連動の設定まで、[セットアップ手順.md](./セットアップ手順.md) にまとめています。所要時間は約15分、費用は無料枠内で収まります。

## 技術構成

| ファイル | 役割 |
|---|---|
| `index.html` | ゲーム本体（HTML/CSS/JSを1ファイルに集約） |
| `取組結果登録ツール.html` | 番付・取組結果・優勝三賞を手動登録して `data/basho.json` を作る管理ツール（公開不要・手元で使用） |
| `scraper/fetch_basho.mjs` | 大相撲ニュース見出しだけをGoogleニュースRSSから取得する軽量スクリプト（任意） |
| `.github/workflows/update-data.yml` | 上記を1日5回実行しニュースのみ更新するGitHub Actions（任意） |
| `data/basho.json` | その場所の実データ（番付・取組結果・優勝三賞。ニュースは自動、それ以外は手動登録） |
| `.nojekyll` | GitHub PagesのJekyllビルドを無効化するための空ファイル |

バックエンドはSupabase（Auth + Postgres）のみ。取組のシミュレーションや判定ロジックはすべてクライアント側の決定論的な処理（シード乱数）で完結しており、専用サーバーは必要ありません。
